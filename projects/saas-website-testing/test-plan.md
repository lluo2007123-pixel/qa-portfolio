# Test Plan – www.rentpotty.com
## Objective
- Validate functionality, usability, and performance to ensure users can place orders, manage
accounts, submit requests, and track services effectively


## Scope
- Functional testing (Order flow, account management, dashboard, service requests, order history, emails)
- Usability testing (UX clarity, friction, navigation)
- Performance testing (speed, responsiveness, reliability)
- Bug testing (UI issues, broken flows, logic errors)

## Test Approach
- Functional, Usability, End-to-End, and Baseline Performance testing using real user scenarios

## Key Test Scenarios
- Order submission 
- Account registration/login 
- Service requests 
- Order tracking 
- Email notifications 
- Performance check

## Tools
- Chrome DevTools
- Lighthouse
- Manual testing
- Postman

## Test Environment
- Browser: Chrome
- Device: Desktop, IPhone 16


## Deliverables
- Test cases 
- Bug report 
- Usability insights 
- Performance observations

## Bug Severity Matrix
- Critical - System crash or core function failure
- High - Major feature not working
- Medium - Partial functionality issue
- Low - Minor UI/UX issue
